Data compression is a fundamental concept in computer science that helps reduce the storage size of files and improves transmission speed over networks.
 Huffman encoding, which is a greedy algorithm, works by assigning shorter binary codes to frequently occurring characters and longer codes to less frequent ones.
 
  This ensures that the overall size of the encoded data is minimized without losing any information. Because of this,
   Huffman encoding is widely used in file compression formats and communication systems.

In a normal text file, some letters like 'e', 't', 'a', 'o', and 'n' appear very frequently, while others like 'z', 'x', and 'q' appear less often.
 Huffman encoding takes advantage of this uneven distribution by creating a binary tree where each character is represented by a unique path.
  The construction of this tree involves repeatedly combining the two least frequent nodes until only one root node remains. From this tree,
   binary codes are generated and used to encode the original data.
